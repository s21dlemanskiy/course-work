# ADR-0002: Хранить полную базу знаний локально

## Статус

Принято

## Дата

2026-06-06

## Контекст

Техник может работать на участках с нестабильной связью. Базовый сценарий обслуживания не должен зависеть от backend: нужно распознать оборудование, найти инструкцию, выполнить чек-лист и сохранить журнал действий. Пользователь уточнил, что локально кэшируются не только открытые инструкции, а вся база знаний.

## Решение

Android Client хранит полную Knowledge Base локально в SQLite или эквивалентном embedded storage. Backend публикует `knowledge_base_version`, а Knowledge Sync Service выдает полную базу знаний или инкрементальные обновления.

`MaintenanceJob` фиксирует `instruction_version`, чтобы начатая операция продолжалась с той инструкцией, с которой была начата.

## Альтернативы

| Альтернатива | Плюсы | Минусы |
|---|---|---|
| Кэшировать только открытые инструкции | Меньше места на устройстве | Офлайн-сценарий непредсказуем: нужной инструкции может не быть |
| Всегда искать инструкции на сервере | Проще актуальность | Базовый сценарий ломается без сети |
| Хранить полную базу знаний локально | Надежный офлайн-сценарий | Нужно управлять размером, версиями и миграциями |

## Последствия

Положительные:

- Техник может работать со всей базой знаний без сети.
- Локальный поиск быстрее и устойчивее к сетевым сбоям.
- Backend может быть временно недоступен без остановки полевых работ.

Отрицательные:

- Нужно контролировать размер Knowledge Base.
- Нужна политика обновлений и совместимости версий.
- Нужно учитывать риск утечки локальной базы знаний.

## Когда пересмотреть

- Если база знаний станет слишком большой для целевых Android-устройств.
- Если требования безопасности запретят хранить полную базу знаний локально.
- Если появится надежная корпоративная сеть на всех участках обслуживания.
