# RMA: Railway Maintenance Assistant

`RMA` (`Railway Maintenance Assistant`) - система помощи техническому персоналу при обслуживании объектов железнодорожной инфраструктуры.

MVP описывает Android-приложение для смартфонов и планшетов, которое распознает оборудование по шильдику, помогает найти инструкцию в локальной базе знаний, провести чек-лист и зафиксировать журнал работ. Серверная часть нужна для управления базой знаний, обновлений, синхронизации журналов и тяжелых AI-функций: LLM/RAG, STT/TTS и расширенного семантического поиска.

## Основные решения MVP

- Целевая платформа - Android-смартфоны и Android-планшеты.
- AR-очки не входят в MVP.
- OCR шильдика выполняется локально на Android-устройстве.
- Полная база знаний кэшируется локально и доступна без сети.
- Чек-листы и журналы действий создаются локально; синхронизация выполняется позже через outbox.
- LLM/RAG, STT/TTS и другие тяжелые модели работают только онлайн на сервере.
- Backend разделен на отдельные сервисы: API Gateway/Auth, Documentation Service, Knowledge Sync Service, Search/RAG Service, Speech Service, Operation Log/Sync Service и Admin Panel.
- EAM-интеграция описана как будущее развитие, не как обязательная зависимость MVP.

## Ключевые сущности и идентификаторы

- `device_id` - стабильный идентификатор оборудования.
- `instruction_version` - версия инструкции или чек-листа.
- `knowledge_base_version` - версия локально кэшируемой базы знаний.
- `client_operation_id` - локальный идентификатор операции обслуживания.
- `operation_event_id` - идентификатор события журнала действий.
- `idempotency_key` - ключ для безопасной повторной синхронизации.
- `correlation_id` - идентификатор для связи логов, метрик и состояния процесса.

## Проверка перед сдачей

- Все Mermaid-диаграммы должны рендериться.
- Названия компонентов и сущностей должны совпадать во всех разделах.
- AR-очки и EAM не должны быть описаны как обязательные зависимости MVP.
- Все существенные архитектурные решения должны быть отражены в ADR.
