# 06. Сценарии и потоки

> Сокращения и рабочие термины расшифрованы в [словаре терминов](13-термины-и-сокращения.md).

## Ручное создание и ведение КСГ

```mermaid
sequenceDiagram
    participant Planner as Планировщик
    participant Web as Web Client
    participant Schedule as Manual Schedule Service
    participant DB as PostgreSQL

    Planner->>Web: Создает проект
    Planner->>Web: Заводит работы, сроки, объемы, пикетаж
    Web->>Schedule: Сохранить работы КСГ
    Schedule->>DB: Записать версию и историю изменений
    Schedule-->>Web: Показать актуальный КСГ
```

## Удаленная проверка через камеры на технике

```mermaid
sequenceDiagram
    participant Equipment as Техника с камерами
    participant Device as Equipment & Camera Service
    participant Inspector as Удаленный инспектор
    participant Web as Web Client
    participant Inspection as Remote Inspection Service
    participant Storage as Object Storage
    participant Schedule as Manual Schedule Service

    Equipment->>Device: Передает координаты и фото/видео
    Inspector->>Web: Открывает технику или работу КСГ
    Web->>Device: Получить камеры и последние координаты
    Device-->>Web: Камеры, координаты, статус сигнала
    Inspector->>Web: Создает вывод проверки
    Web->>Inspection: Сохранить remote_inspection
    Inspection->>Storage: Сохранить доказательные материалы
    Inspection-->>Schedule: Ссылка на проверку для работы
    Inspector->>Web: Вручную меняет статус работы при необходимости
    Web->>Schedule: Сохранить ручное изменение КСГ
```

## Жизненный цикл WorkItem

```mermaid
stateDiagram-v2
    direction LR
    [*] --> Draft
    Draft --> Planned
    Planned --> InProgress
    InProgress --> NeedsReview
    NeedsReview --> InProgress
    InProgress --> Done
    Done --> Accepted
    Accepted --> Closed
    Planned --> Cancelled
    InProgress --> Cancelled
```

| Переход | Условие |
|---|---|
| `Draft -> Planned` | Планировщик добавил работу в КСГ |
| `Planned -> InProgress` | Пользователь вручную отметил старт |
| `InProgress -> NeedsReview` | Удаленный инспектор создал замечание |
| `NeedsReview -> InProgress` | Замечание принято в работу |
| `InProgress -> Done` | Генподрядчик сообщил о готовности |
| `Done -> Accepted` | Инспектор или заказчик принял работу |
| `Accepted -> Closed` | Работа закрыта в отчетном периоде |
| `Planned/InProgress -> Cancelled` | Пользователь отменил или заменил работу |

## Правила повторов

- Запись удаленного инспектора передает `client_event_id`; повторная отправка не создает дубль.
- Медиафайл имеет checksum; повторная загрузка связывается с той же проверкой.
- GNSS-точки дедуплицируются по `equipment_id`, времени и sequence number, если он доступен.
- Ручное изменение КСГ создает отдельную запись истории и не перезаписывает старую без следа.

## Ошибочные ветки

| Ошибка | Реакция |
|---|---|
| Камера недоступна | Показать статус нет сигнала и дать создать замечание |
| GNSS не передает координаты | Показать последнюю координату и время сигнала |
| Инспектор не уверен в факте | Создать статус `needs_on_site_check` или `needs_clarification` |
| Пользователь без прав пытается менять КСГ | Запретить действие и записать security event |
| Медиафайл не загрузился | Не принимать запись проверки как полную, предложить повтор |
| Future-импорт документов еще не реализован | Документ хранится как вложение без автоматического создания работ |
