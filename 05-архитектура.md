# 05. Архитектура

## Архитектурный стиль

RMA строится как offline-capable Android Client и набор специализированных backend-сервисов. Android Client является основным рабочим инструментом техника и хранит локальный источник для полевого сценария: полную базу знаний, OCR-модуль, чек-листы, локальный журнал и outbox. Backend отвечает за публикацию знаний, синхронизацию, администрирование и тяжелые онлайн-функции.

## C4 Container

```mermaid
C4Container
    title Контейнерная диаграмма RMA MVP

    Person(technician, "Техник", "Выполняет обслуживание")
    Person(admin, "Администратор базы знаний", "Управляет контентом")

    System_Boundary(rma, "RMA") {
        Container(android, "Android Client", "Android/Kotlin", "OCR, локальная база знаний, чек-листы, журнал, outbox")
        Container(adminPanel, "Admin Panel", "Web UI", "Интерфейс администратора")
        Container(gateway, "API Gateway/Auth", "Backend service", "Аутентификация, маршрутизация, единая входная точка API")
        Container(doc, "Documentation Service", "Backend service", "Устройства, инструкции, чек-листы, версии контента")
        Container(sync, "Knowledge Sync Service", "Backend service", "Полная и инкрементальная выдача базы знаний клиентам")
        Container(search, "Search/RAG Service", "Backend service", "Онлайн-RAG и семантический поиск")
        Container(speech, "Speech Service", "Backend service", "Онлайн STT/TTS")
        Container(logsync, "Operation Log/Sync Service", "Backend service", "Прием outbox-событий и журналов работ")
        ContainerDb(db, "PostgreSQL", "Database", "Контент, версии, пользователи, операции")
        ContainerDb(vector, "Vector Index", "Vector storage", "Индекс для RAG и семантического поиска")
        ContainerDb(files, "Object Storage", "Storage", "Фото и другие вложения")
        Container(queue, "Message Broker", "Queue", "Фоновая индексация, публикация версий, обработка вложений")
    }

    Rel(technician, android, "")
    Rel(admin, adminPanel, "")
    Rel(android, gateway, "HTTPS")
    Rel(adminPanel, gateway, "HTTPS")
    Rel(gateway, doc, "")
    Rel(gateway, sync, "")
    Rel(gateway, search, "")
    Rel(gateway, speech, "")
    Rel(gateway, logsync, "")
    Rel(doc, db, "")
    Rel(sync, db, "")
    Rel(search, db, "")
    Rel(search, vector, "")
    Rel(logsync, db, "")
    Rel(logsync, files, "")
    Rel(doc, queue, "")
    Rel(queue, search, "")

    UpdateLayoutConfig($c4ShapeInRow="3", $c4BoundaryInRow="1")
```

На этой диаграмме намеренно не показан `Local SQLite`: это встроенное хранилище внутри Android Client, а не самостоятельный контейнер уровня RMA. Его ответственность раскрыта на component-диаграмме Android Client и в разделе данных.

| Откуда | Куда | Назначение |
|---|---|---|
| Android Client | API Gateway/Auth | Синхронизация outbox, обновление базы знаний, онлайн-RAG и Speech |
| Admin Panel | API Gateway/Auth | Администрирование устройств, инструкций и версий |
| API Gateway/Auth | Backend-сервисы | Авторизованная маршрутизация запросов |
| Documentation Service | PostgreSQL, Message Broker | Хранение контента и события публикации |
| Knowledge Sync Service | PostgreSQL | Чтение опубликованных версий базы знаний |
| Search/RAG Service | PostgreSQL, Vector Index | Онлайн-поиск по опубликованной базе знаний |
| Operation Log/Sync Service | PostgreSQL, Object Storage | Идемпотентный прием журналов и вложений |

## Ответственность контейнеров

| Контейнер | Ответственность |
|---|---|
| Android Client | Основной UI техника, локальный OCR, локальный поиск, чек-листы, журналы, outbox |
| API Gateway/Auth | Авторизация, проверка токенов, маршрутизация API, выдача `correlation_id` |
| Documentation Service | Управление устройствами, инструкциями, чек-листами и версиями |
| Knowledge Sync Service | Выдача полной базы знаний и инкрементальных обновлений |
| Search/RAG Service | Онлайн-помощь, семантический поиск, RAG по опубликованной базе знаний |
| Speech Service | STT/TTS для голосового ввода и голосовых подсказок при наличии сети |
| Operation Log/Sync Service | Прием outbox-событий, идемпотентность, сохранение журналов |
| Admin Panel | Пользовательский интерфейс администратора |

## C4 Component: Android Client

```mermaid
C4Component
    title Компоненты Android Client

    Container_Boundary(android, "Android Client") {
        Component(ui, "Maintenance UI", "Android UI", "Сканирование, инструкции, чек-листы, статусы")
        Component(ocr, "Local OCR Module", "On-device ML", "Извлекает маркировку со шильдика")
        Component(searchLocal, "Local Search", "Kotlin module", "Ищет по локальной базе знаний")
        Component(checklist, "Checklist Manager", "Kotlin module", "Ведет шаги и подтверждения")
        Component(jobStore, "Job Store", "Repository", "Хранит MaintenanceJob и OperationLog")
        Component(kbStore, "Knowledge Base Store", "Repository", "Хранит полную базу знаний")
        Component(localStore, "Local Storage", "SQLite", "Knowledge Base, MaintenanceJob, OperationLog, outbox")
        Component(outbox, "Outbox Sync", "Worker", "Отправляет события на сервер с идемпотентностью")
        Component(online, "Online Assist Adapter", "HTTP client", "Вызывает RAG и Speech через Gateway")
    }

    Rel(ui, ocr, "Сканирует шильдик")
    Rel(ui, searchLocal, "Ищет инструкцию")
    Rel(ui, checklist, "Выполняет чек-лист")
    Rel(checklist, jobStore, "Сохраняет состояние")
    Rel(searchLocal, kbStore, "Читает знания")
    Rel(jobStore, localStore, "")
    Rel(kbStore, localStore, "")
    Rel(outbox, jobStore, "Читает несинхронизированные события")
    Rel(ui, online, "Онлайн-подсказки при наличии сети")
```

## Ключевые политики

| Политика | Где реализуется | Почему здесь | Как проверить |
|---|---|---|---|
| Версионирование базы знаний | Documentation Service, Knowledge Sync Service, Android Client | Клиент должен знать, какую версию хранит локально | Contract и integration tests |
| Фиксация версии инструкции в операции | Checklist Manager, Job Store | Операция должна продолжаться с той инструкцией, с которой началась | E2E-тест обновления во время операции |
| Идемпотентность outbox | Android Client, Operation Log/Sync Service | Сеть может обрываться, события могут отправляться повторно | Failure test |
| Fallback онлайн-функций | Android Client | Техник не должен терять базовый сценарий при отказе RAG/STT/TTS | E2E-тест отказа сервисов |
| Ownership checks | API Gateway/Auth, Operation Log/Sync Service | Техник не должен видеть чужие операции | Security tests |
