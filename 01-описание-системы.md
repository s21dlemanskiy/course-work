# 01. Описание системы

## Назначение

`RMA` (`Railway Maintenance Assistant`) - мобильная система помощи техническому персоналу при обслуживании железнодорожного оборудования. Система помогает технику определить устройство по шильдику, найти актуальную инструкцию в базе знаний, выполнить чек-лист и сохранить цифровой журнал операции.

Главная проблема: в полевых условиях техник тратит время на поиск бумажной или разрозненной электронной документации, рискует открыть устаревшую инструкцию и не всегда может сразу зафиксировать ход работ. RMA снижает время поиска решения и делает выполнение операции воспроизводимым.

## Целевые пользователи

| Пользователь | Потребность |
|---|---|
| Техник | Быстро определить оборудование, найти инструкцию, выполнить чек-лист и зафиксировать результат работ |
| Обучающийся сотрудник | Проходить операции по пошаговой инструкции и видеть подсказки |
| Администратор базы знаний | Загружать устройства, инструкции, чек-листы и версии базы знаний |
| Инженер эксплуатации RMA | Контролировать синхронизацию, ошибки, версии базы знаний и состояние сервисов |

## Основные сценарии MVP

- Техник открывает Android-приложение, сканирует шильдик и получает `device_id`.
- Приложение ищет устройство и инструкции в локальной полной базе знаний.
- Техник выбирает или уточняет проблему текстом; при наличии сети может использовать онлайн-помощь LLM/RAG.
- Техник выполняет чек-лист и подтверждает шаги.
- Приложение сохраняет `MaintenanceJob` и `OperationLog` локально.
- При появлении сети outbox синхронизирует журналы и получает обновления базы знаний.
- Администратор через Admin Panel публикует новую версию базы знаний.

## Границы MVP

В MVP входит:

- Android-приложение для смартфонов и планшетов;
- локальный OCR шильдиков;
- локальная полная база знаний;
- локальный поиск по устройствам, инструкциям и чек-листам;
- локальное ведение журналов действий;
- outbox-синхронизация журналов;
- серверное управление базой знаний;
- онлайн-функции LLM/RAG и STT/TTS как расширение сценария при наличии сети.

В MVP не входит:

- поддержка AR-очков;
- обязательная интеграция с EAM-системой;
- автоматическое управление ремонтными заданиями во внешних системах;
- гарантия работы LLM/RAG/STT/TTS без сети;
- распознавание любых объектов без шильдика и справочника оборудования.

## Критерии успеха MVP

| Критерий | Как проверить |
|---|---|
| Техник может выполнить операцию без сети | Демонстрация: OCR, локальный поиск, чек-лист и журнал работают с полной локальной базой знаний |
| Инструкции обновляются централизованно | Администратор публикует новую `knowledge_base_version`, клиент скачивает обновление |
| Повторная синхронизация не создает дубли | Повторная отправка `OperationLog` с тем же `idempotency_key` не меняет состояние дважды |
| Онлайн-помощь деградирует безопасно | При недоступности RAG или Speech сервисов остается локальный поиск и ручной ввод |
| Документация связана с архитектурой | Требования, диаграммы, данные, тесты и ADR не противоречат друг другу |

## Схема ценности

```mermaid
flowchart LR
    Technician[Техник] --> Problem[Нужно быстро обслужить оборудование]
    Problem --> RMA[RMA Android Client]
    RMA --> OCR[Локальное распознавание шильдика]
    OCR --> KB[Локальная база знаний]
    KB --> Checklist[Чек-лист и инструкция]
    Checklist --> Log[Цифровой журнал работ]
```

## Допущения

- Android-устройство имеет камеру, достаточно памяти для полной базы знаний и защищенное локальное хранилище.
- База знаний обновляется пакетами или инкрементально по версии.
- В MVP локальная база знаний считается достаточной для типовых операций, а онлайн-RAG используется как дополнительная помощь.
